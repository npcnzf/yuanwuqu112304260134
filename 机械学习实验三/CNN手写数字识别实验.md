## 实验一：模型训练与超参数调优（必做）

### 1.1 实验目标

### 1.2 模型结构（统一）

所有实验使用以下基础结构：

```
输入(1×28×28) → Conv1 + ReLU + MaxPool → Conv2 + ReLU + MaxPool → Flatten → FC → 输出(10类)
```

**详细结构**：
- Conv1: 32个3×3卷积核，ReLU激活，2×2最大池化
- Conv2: 64个3×3卷积核，ReLU激活，2×2最大池化
- Dropout(0.25)
- FC1: 128神经元，ReLU激活，Dropout(0.5)
- FC2: 10神经元，LogSoftmax输出

| Exp2 | Adam | 0.001 | 64 | 否 | 否 |
| Exp3 | Adam | 0.001 | 128 | 否 | 是 |
| Exp4 | Adam | 0.001 | 64 | 是 | 是 |

### 1.6 分析问题（请逐条回答）

**Q1：Adam 和 SGD 的收敛速度有何差异？从实验结果中你观察到了什么？**

Adam 优化器的收敛速度明显快于 SGD。从实验结果可以看出：
- Exp1（SGD）在50个Epoch后训练准确率才达到99.92%，且验证准确率相对较低（98.65%）
- Exp2（Adam）在20个Epoch左右就达到了较高的准确率，最终训练准确率达到99.95%，验证准确率达到98.92%
- Adam 通过自适应学习率调整，能够在训练初期快速下降，而SGD需要更多迭代才能达到相似效果

**Q2：学习率对训练稳定性有什么影响？**

学习率直接影响训练的稳定性和收敛速度：
- 学习率过大（如SGD使用0.01）会导致Loss曲线震荡，可能无法收敛到最优解
- 学习率过小会导致收敛速度过慢，需要更多训练时间
- Adam使用较小的学习率（0.001）能够稳定收敛，验证集表现更好
- 合适的学习率能够平衡收敛速度和稳定性，避免过拟合或欠拟合

**Q3：Batch Size 对模型泛化能力有什么影响？**

Batch Size 对模型泛化能力有显著影响：
- Exp2（Batch Size=64）验证准确率98.92%
- Exp3（Batch Size=128）验证准确率99.01%
- 较大的Batch Size使梯度估计更稳定，减少噪声，有助于模型学习更通用的特征
- 但过大的Batch Size可能导致过拟合，因为每个批次包含的样本多样性减少
- 本实验中，Batch Size=128在验证集上表现略好于Batch Size=64

**Q4：Early Stopping 是否有效防止了过拟合？**

是的，Early Stopping 有效防止了过拟合：
- Exp3和Exp4使用了Early Stopping，分别在32和28个Epoch提前终止
- Exp3的验证准确率（99.01%）高于Exp2（98.92%），尽管训练准确率稍低（99.88% vs 99.95%）
- Exp4的验证准确率最高（99.23%），说明Early Stopping成功防止了过拟合
- 没有Early Stopping的Exp1和Exp2在后期可能出现过拟合现象

**Q5：数据增强是否提升了模型的泛化能力？为什么？**

是的，数据增强显著提升了模型的泛化能力：
- Exp4（使用数据增强）的验证准确率（99.23%）明显高于Exp2（98.92%）
- 数据增强通过随机旋转、平移等操作增加了训练数据的多样性
- 这迫使模型学习更鲁棒的特征表示，而不是记忆训练样本
- 增强后的模型能够更好地应对测试集中的微小变化，提高了泛化能力
